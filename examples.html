Transform

  1. translate
  2. skew
  3. scale
  4. rotate
  5. perspective
  6. matrix
  7. 3D
translate
Specifies a 2D translation by vector [tx, ty]
translate(tx[, ty]);
if ty isn't specified, its value is assumed zero
translateX(ax)
translates amount along X axis
translateY(ay)
translates amount along Y axis
skew
Specifies a 2D translation by vector [tx, ty]
skew(ax[, ay]);
if ay isn't specified, no skew happens on the Y axis
skewX(angle)
skews amount along X axis by given angle
skewY(angle)
skews amount along Y axis by given angle
scale
Specifies a 2D scaling operation with a unitless number
scale(sx[, sy]);
if sy isn't specified, no scaling happens on the Y axis'
scaleX(sx)
skews amount along X axis by given angle
scaleY(sy)
skews amount along Y axis by given angle
rotate
Rotates element clockwise around its origin by the specified angle
rotate(angle)
can be negative
perspective
Distance between the Z plane and the user to give a 3D perspective. The vanishing point is placed in the center by default, but can be changed using perspective-origin
Applying a perspective to a parent element will give the same perspective point for all its children. See Parents and Perspective
perspective(length)
length is the distance measured away from Z plane
perspective-origin: % | position
lengths are ok. See Perspective-Origin Values
matrix
Builds transformation matrix with translate values
matrix(a, b, c, d, tx, ty)
a: top-left , b: bottom-left , c: top-right , d: bottom-left , tx: translateX , ty: translateY
perspective-origin: % | position
lengths are ok. See Perspective-Origin Values

Transition

1. Single iteration
2. Requires CSS property being changed
3. Transition:all (could be bad for performance)
4. Best paired with javaScript
transition-delay
Configures delay before the transition occurs once activated
transition-duration
Configures how long the transition will happen until it achieves the new property
transition-property
Specifies which properties will be transitioned. The browser listens for the exact ones explicitly stated, unless 'all' is declared
transition-timing-function
Configures how the transition gets to the next value. Usually a curve in time

Animations

1. Infinite iterations
2. Doesn't require triggers
3. Multiple intermediate points
animation-delay
Configures the delay between the time the element is loaded and the beginning of the animation sequence.
[ 3s ]
indicates seconds
[ 2s, 4ms ]
2:04 seconds
animation-direction
Configures whether or not the animation should alternate direction on each run through the sequence or reset to the start point and repeat itself.
[ normal ]
plays normally
[ reverse ]
plays backwards
[ alternate ]
alternates between forward and backward timing functions included
[ alternate-reverse ]
alternates starting with reversed
animation-duration
Configures the length of time that an animation should take to complete one cycle.
[ 6s ]
6 seconds
[ 120ms ]
120 milliseconds
animation-iteration-count
Configures the number of times the animation should repeat; you can specify infinite to repeat the animation indefinitely.
[ infinite ]
plays foreverrrr
[ 3 ]
repeats 3 times
[ 2.3 ]
repeats 2 times, then for .3 of a third iteration
animation-name
Specifies the name of the @keyframes at-rule describing the animation's keyframes.
[ none ]
used to deactivate an animation without changing order of the other identifiers
[ IDENT ]
case-insensitive identifier(s) listed by commas.
[ 2.3 ]
repeats 2 times, then for .3 of a third iteration
animation-play-state
Lets you pause and resume the animation sequence.
W3C is considering removing this property
[ running ]
animation is currently playing
[ paused ]
animation is currently paused
[ paused, running, running ]
  what does this mean?!
animation-timing-function
Configures the timing of the animation; that is, how the animation transitions through keyframes, by establishing acceleration curves.
See: Bezier Curve Comparison
animation-fill-mode
Configures what values are applied by the animation before and after it is executing.
[ none ]
Target will not apply any styles to the target before or after it is executing
[ forwards ]
Target will retain the computed values set by the last keyframe encountered during execution (retained during animation-delay)
[ backwards ]
The animation will apply values defined in the first relevant keyframe as soon as it is applied to the target (retained during animation-delay).
