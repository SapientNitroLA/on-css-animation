CSS Animateable Properties

animation-delay
Configures the delay between the time the element is loaded and the beginning of the animation sequence.
[ 3s ]
indicates seconds
[ 2s, 4ms ]
2:04 seconds
animation-direction
Configures whether or not the animation should alternate direction on each run through the sequence or reset to the start point and repeat itself.
[ normal ]
plays normally
[ reverse ]
plays backwards
[ alternate ]
alternates between forward and backward timing functions included
[ alternate-reverse ]
alternates starting with reversed
animation-duration
Configures the length of time that an animation should take to complete one cycle.
[ 6s ]
6 seconds
[ 120ms ]
120 milliseconds
animation-iteration-count
Configures the number of times the animation should repeat; you can specify infinite to repeat the animation indefinitely.
[ infinite ]
plays foreverrrr
[ 3 ]
repeats 3 times
[ 2.3 ]
repeats 2 times, then for .3 of a third iteration
animation-name
Specifies the name of the @keyframes at-rule describing the animation's keyframes.
[ none ]
used to deactivate an animation without changing order of the other identifiers
[ IDENT ]
case-insensitive identifier(s) listed by commas.
[ 2.3 ]
repeats 2 times, then for .3 of a third iteration
animation-play-state
Lets you pause and resume the animation sequence.
W3C is considering removing this property
[ running ]
animation is currently playing
[ paused ]
animation is currently paused
[ paused, running, running ]
  what does this mean?!
animation-timing-function
Configures the timing of the animation; that is, how the animation transitions through keyframes, by establishing acceleration curves.
See: Bezier Curve Comparison
animation-fill-mode
Configures what values are applied by the animation before and after it is executing.
[ none ]
Target will not apply any styles to the target before or after it is executing
[ forwards ]
Target will retain the computed values set by the last keyframe encountered during execution (retained during animation-delay)
[ backwards ]
The animation will apply values defined in the first relevant keyframe as soon as it is applied to the target (retained during animation-delay).
